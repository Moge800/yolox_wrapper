# GUI ガイド

YOLOX Wrapper には tkinter ベースの GUI アプリケーションが付属しています。

## 起動方法

```bash
uv run yolox-gui
# または
uv run python main.py
```

## 画面構成

アプリケーションは以下の要素で構成されています。

- **メニューバー** — ファイル（設定の保存）とプロファイル（追加・削除）操作
- **プロファイルセレクター** — 上部のドロップダウンで設定プロファイルを切り替え
- **タブ** — 学習・推論・カメラ・エクスポートの 4 タブ

---

## 学習タブ (Train)

YOLOX モデルの学習を行います。

### 入力項目

| 項目 | 説明 |
|------|------|
| data.yaml | Label Studio COCO エクスポート設定ファイルのパス |
| モデルサイズ | `nano` / `tiny` / `s` / `m` / `l` / `x` から選択 |
| エポック | カンマ区切りの段階的スケジュール（例: `100,200,300`） |
| バッチサイズ | 学習バッチサイズ |
| 画像サイズ | 入力画像の辺長（32〜1920、32 刻み） |
| ワーカー数 | データ読み込みの並列数 |
| 検証分割率 | 検証データの割合（0.05〜0.5） |
| デバイス | CPU または GPU (cuda:0) |

### 操作

1. 各項目を設定
2. **Start** ボタンで学習開始（バックグラウンドスレッドで実行）
3. ログエリアにリアルタイムで学習経過が表示
4. 進捗バーで段階（ステージ）ごとの進行状況を確認
5. **Stop** ボタンで安全に中断可能

> 学習完了時にビープ音で通知されます。

---

## 推論タブ (Infer)

学習済みモデルで単一画像の推論を行います。

### 入力項目

| 項目 | 説明 |
|------|------|
| モデルパス | `.pt` ファイルのパス |
| 画像パス | 推論対象の画像ファイルまたはディレクトリ |
| 信頼度しきい値 | 検出の信頼度フィルター（0.01〜1.0） |
| IoU しきい値 | NMS の IoU フィルター（0.01〜1.0） |
| デバイス | CPU または GPU |

### 操作

1. モデルと画像を選択
2. しきい値を調整
3. 推論実行で結果画像と検出リストが表示

> モデルは遅延読み込みされ、パスが変更された場合のみ再読み込みされます。

---

## カメラタブ (Camera)

USB カメラからのリアルタイム物体検出を行います。

### 入力項目

| 項目 | 説明 |
|------|------|
| モデルパス | `.pt` ファイルのパス |
| カメラ番号 | カメラデバイスインデックス（0〜9） |
| 信頼度しきい値 | 検出の信頼度フィルター |
| IoU しきい値 | NMS の IoU フィルター |
| デバイス | CPU または GPU |

### 操作

1. モデルとカメラ番号を設定
2. **Start** でライブ推論開始
3. FPS がリアルタイムで表示
4. **Stop** で停止

> 約 30fps を目標にフレームが更新されます。

---

## エクスポートタブ (Export)

学習済みモデルを ONNX 形式にエクスポートします。

### 入力項目

| 項目 | 説明 |
|------|------|
| 入力モデル | `.pt` ファイルのパス |
| 出力パス | `.onnx` ファイルの保存先（省略時は自動設定） |

### 操作

1. 入力モデルを選択（出力パスが自動で候補表示）
2. **Export** ボタンでエクスポート実行
3. ステータスが緑（成功）または赤（エラー）で表示

---

## プロファイル管理

- **プロファイル追加**: メニュー → Profile → Add Profile でプロファイル名を入力
- **プロファイル削除**: メニュー → Profile → Remove Profile で現在のプロファイルを削除
- **設定保存**: メニュー → File → Save Config で現在の全タブ設定を `config.ini` に保存

> `default` プロファイルは削除できません。
